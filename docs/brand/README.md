## Current logo

The current logo lives at [`apps/customer/public/brand/logo.svg`](../../apps/customer/public/brand/logo.svg). At runtime on the customer app it serves as `/brand/logo.svg`. The file is the canonical, evolving brand asset: future iterations replace it in place. Historical drafts live in Git history.

The current iteration was generated by Recraft from [`logo-prompts.md`](logo-prompts.md) (direction B: mark-led + wordmark below) and retinted to the bibs palette. Recraft's chosen purple `#4746DA` was mapped to **Ink** (`#2f3f76` ≈ `oklch(0.34 0.10 258)`) and Recraft's pink `#F6C7FF` to **Cream** (`#fbf8f0` ≈ `oklch(0.985 0.008 80)`).

It is **not the final mark**. It is one iteration on the route to it. Open improvements:

- The square Ink-on-Cream lockup works as an app icon, favicon, social card. A transparent-background variant (mark-only on a Cream surface) and a knockout variant (Cream mark on Ink hero) are still missing.
- The wordmark inside the lockup is a Recraft custom display, not yet aligned with Bricolage Grotesque. A type-tuned final wordmark (custom or Bricolage-based) is the next step.
- No separate mark-only file (just the open hand, no wordmark) for tight-square contexts.
- Pixel-perfect color verification in a color-managed tool is pending; the hex values are sRGB approximations of the canonical OKLCH definitions in DESIGN.md frontmatter.

## Direction selected

**B. Mark-led + wordmark below.** The open-palm symbol is the primary mark; "bibs" wordmark sits below in lockup. Reasons in [`logo-brief.md`](logo-brief.md) §3.

|---|---|---|

| Cream | `oklch(0.985 0.008 80)` | `#fbf8f0` |
| Saffron | `oklch(0.78 0.14 75)` | `#e6b057` |

OKLCH is canonical. Hex is an sRGB approximation.

## Workflow for iterating

1. Open the SVG drafts directly in a browser to evaluate the geometry.
2. For fast exploration, run [`logo-prompts.md`](logo-prompts.md) in Recraft or Midjourney.
3. For a polished finale, send [`logo-brief.md`](logo-brief.md) to a designer. Brief is self-contained.
4. When a new mark is approved, replace `apps/customer/public/brand/logo.svg` with the new file and update `DESIGN.md` if any tokens shift.
